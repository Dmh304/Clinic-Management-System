- Đã thêm Default Constraint mới cho cột priority mặc định là 'PRIMARY' (Constraint Name: DF_lab_orders_priority).

ii) Cập nhật Trạng thái phiếu xét nghiệm (Status):
- Đã chạy script xoá Check Constraint cũ giới hạn trạng thái (Constraint Name: CK_lab_orders_status).
- Đã cập nhật toàn bộ dữ liệu status cũ trong bảng lab_orders của các cơ sở dữ liệu:
  + Chuyển đổi trạng thái cũ 'COMPLETED' thành 'APPROVED'
  + Chuyển đổi trạng thái cũ 'CANCELLED' thành 'REJECTED'
  để tương thích với trạng thái hoạt động của code nghiệp vụ.
- Đã thêm Check Constraint mới CK_lab_orders_status giới hạn trong 5 giá trị: 'PENDING', 'IN_PROGRESS', 'SUBMITTED', 'REJECTED', 'APPROVED'.
- Câu lệnh SQL đã thực thi cho phần Status:
  --------------------------------------------------
  USE ecms_db_final; -- Áp dụng tương tự cho ecms_backup và ecms_db
  GO

  IF EXISTS (SELECT 1 FROM sys.check_constraints WHERE name = 'CK_lab_orders_status' AND parent_object_id = OBJECT_ID('lab_orders'))
  BEGIN
      ALTER TABLE lab_orders DROP CONSTRAINT CK_lab_orders_status;
  END

  UPDATE lab_orders SET status = 'APPROVED' WHERE status = 'COMPLETED';
  UPDATE lab_orders SET status = 'REJECTED' WHERE status = 'CANCELLED';

  ALTER TABLE lab_orders ADD CONSTRAINT CK_lab_orders_status CHECK (status IN ('PENDING', 'IN_PROGRESS', 'SUBMITTED', 'REJECTED', 'APPROVED'));
  GO
  --------------------------------------------------
